RPC接口测试框架
(包含用例管理后台+在线报告)

该框架主要用于加密货币量化交易系统 RPC 接口的自动化测试
第一版本用例使用 Tablib + Excel 管理，后改为 MySQL 管理，使用 Django Admin 开发了对应的用例管理后台
Pytest + Allure + Gitlab CI + Docker 实现每日自动化测试，并生成在线测试报告

测试数据构建平台

该框架主要用于产研测人员构建一些日常测试数据，极大的提升了测试和验收的效率
1. 免除重复登录服务器和访问数据库等操作
2. 替代 QA 同学手工创建测试数据的繁琐步骤
3. 方便没有权限或不懂 SQL 和 Shell 的PM同学使用
4. 集成一些测试小工具方便验收：比如生成任意字数文本和任意尺寸的图片等
5. 支持一键触发接口自动化测试框架流水线等功能

可将常用的 Shell 脚本、SQL语句、自定义的API脚本等配置到后台，然后通过前段界面快捷操作使用
涉及的第三方库有: mysqlclient, redis, paramiko, pillow, faker等

HTTP 接口测试框架

该框架主要用于HTTP(S)接口自动化回归测试，适用于公司全业务线
使用 Excel、Yaml、Json 分别管理用例和配置数据等。
支持用例录制：将Charles等抓包工具导出的通用HAR格式接口数据，自动转换为可执行用例，极大的提高了用例编写效率
用例分层，独立且可复用性高，支持多接口间调用，支持多环境随意切换，支持自定义函数等。
封装了DeepDiff库，可针对返回结果做字段是否缺失、类型是否正确、返回值是否匹配等的断言和校验、以此生成测试结果和报告。
并加入到了GitLab CI流水线中，部署在服务器Docker环境下，实现持续集成，钉钉消息通知。
